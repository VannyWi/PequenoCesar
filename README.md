# Sistema Integral de Inventario y Gestión de Pedidos - Little Caesars

## 🍕 Sobre el Proyecto
Este proyecto propone el desarrollo de un Sistema Web Integral de Inventarios y Gestión de Pedidos, diseñado para optimizar el flujo de abastecimiento de las tiendas Little Caesars. El objetivo es digitalizar el flujo operativo, conectando las ventas en tiempo real con el control de almacenes, para garantizar la promesa "Hot-N-Ready" eliminando quiebres de stock y errores de registros manuales.

## 🚀 Características Principales
* **Autenticación y Roles:** Login seguro con contraseñas cifradas con BCrypt y gestión de roles (Gerente, Empleado).
* **Módulo de Inventario:** Gestión de insumos con actualización automática al confirmar ventas o recepción de pedidos.
* **Sistema de Alertas:** Notificaciones visuales automáticas cuando los insumos alcanzan el stock mínimo o están próximos a vencer, reduciendo las mermas económicas.
* **Gestión de Pedidos a Proveedores:** Creación rápida de pedidos sugeridos por el sistema basándose en el consumo histórico, con seguimiento de estados (pendiente, en camino, recibido).
* **Dashboard Analítico de KPIs:** Panel en tiempo real que muestra el stock total, alertas activas, porcentaje de merma y gráficos de consumo semanal.

## 🛠️ Stack Tecnológico
* **Backend:** Spring Boot (Java) para la creación de la API REST.
* **Base de Datos:** PostgreSQL, gestionada mediante pgAdmin.
* **Frontend:** Construido con HTML5, CSS3 y JavaScript.
* **Infraestructura:** Docker para la contenedorización de la aplicación y base de datos.
* **Control de Versiones y Pruebas:** Git, GitHub y Postman.
